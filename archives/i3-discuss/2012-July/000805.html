I hit the same problem and found three ways to solve it:

1. Downgrade to xf86-video-intel-2.18.0-3. There is no need to
downgrade kernel; all you need is rebuild the package (i.e. via ABS) to
link it against new versions of libraries. Note that occasionaly you'll
have to rebuild it again after system upgrades.

2. Downgrade to cairo-1.10.2-3 (possibly newer version prior to
1.12.2-2 will do). I tried it after reading comments to bug
https://bugs.freedesktop.org/show_bug.cgi?id=48395 and it also solved
the problem (without downgrading xf86-video-intel).

3. Replace xf86-video-intel-uxa with xf86-video-intel-sna. This may not
work for every platform, but this solution seems to be the most
convenient since it doesn't require keeping outdated packages in the
system.

Hope it hepls.
