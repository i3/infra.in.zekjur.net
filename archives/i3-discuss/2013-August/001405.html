"amixer set Master 1%+" increases the volume linearly. But the 
characteristic line of the human ear is a logarithmic funtion. "-M" 
Tries to compensate this to achieve a felt linear increasment of volume.

> missed the fact that the volume keys are not working out of the box, 
> and turns out it is quite simple to achieve that, so I wonder if it 
> can be added to the initial config script, as it can help users to 
> don't bother about that small useful feature.
>
> I added in my config the option to raise and low the volume with 
> amixer, I'm sure there must be a better way to do that, so if anyone 
> knows better ... share it!
>
> # Section to control the Master output
> bindSym XF86AudioRaiseVolume exec amixer set Master 1%+
> bindSym XF86AudioLowerVolume exec amixer set Master 1%-
> bindSym XF86AudioMute exec amixer -D pulse set Master 1+ toggle
>
> #Section to control the Capture (mic)
> bindSym Shift+XF86AudioRaiseVolume exec amixer set Capture 1%+
> bindSym Shift+XF86AudioLowerVolume exec amixer set Capture 1%-
> bindSym Shift+XF86AudioMute exec amixer set Capture 1+ toggle
