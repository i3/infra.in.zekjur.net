Hi

i3-dmenu-desktop was always a bit slow for me so one day I rewrote it
in C/++. In the last few weeks I worked on it again for a few hours or
two and now I think it can actually be used by people...

For compiling from source it doesn't require anything except cmake and
a compiler with some support for C++11, not much though, so GCC 4.6
probably works, too.

It doesn't support the --entry-type option i3-dmenu-desktop has, but
allows overriding the terminal emulator directly (--term)

Here you go: https://github.com/enkore/j4-dmenu-desktop
