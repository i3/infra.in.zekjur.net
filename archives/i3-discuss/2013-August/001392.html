See http://i3-wm.org/docs/i3bar-protocol.html#_click_events for info
about click events.

Click events basically allow i3status-like programs (i3pystatus uses
this) to react to user input. For example in i3pystatus you can click
on the MPD block to play/pause.

When making the patch back then I was not aware of that feature and
nobody did notice it. Seems like there wasn't a test for it.
