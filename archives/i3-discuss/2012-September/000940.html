> This is currently not supported AFAIK, but if you're interested in

As I said, it is. Carlos is talking changing focus via keyboard, not
via mouse.

>> First, when there are two floating windows one on top of another,
>> changing focus between them does not bring the lower one to the front
>> unless I click on it with the mouse. Is there any way to automatically
>> follow a 'change focus' action with a 'bring to front' action? I can't
>> find a command for the second one.
Patch welcome. I think raising a floating window on focus should be the
default action, so no new configuration directives are required.

>> Second, what is the closest method to get a floating layout to an
>> entire workspace (and one that you can enable/disable dynamically)?
I’m not sure what exactly you mean by that. Making all windows on one
workspace floating is not supported, also marking workspaces as
floating-by-default is not supported. i3 is primarily a tiling window
manager, floating windows are only for popup windows.
