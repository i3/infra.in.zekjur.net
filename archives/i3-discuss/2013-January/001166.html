Marcelo Cerri <mhcerri at gmail.com> writes:
> Is there any chance of some variation of it be accepted? Maybe a new module
> just for that? The existing "time" module would remain intact (maybe
> sharing some code with the new module) and the new module would always
> require a timezone, maybe as its title. It can be named "tz_time" or

Using a new module is definitely the way to go. Nevertheless, I don’t
really want to add such a feature. As I mentioned in the review, it’s
_trivial_ to achieve this with a simple wrapper script, after all you
just need a “TZ=foo date” call. Having it in i3status itself is far more
complex and I’m not convinced that this is worth the effort.
