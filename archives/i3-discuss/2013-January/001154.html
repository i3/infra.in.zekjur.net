> appeared), it resizes to the "want have"-size (or some intermediate

That sounds like an interesting idea. We do call our client-window
buckets 'containers', so there's nothing that stops someone from going
off and implementing something like CSS width/height specifiers with
values that are either (a) a percentage of the screen size, or (b) a
percentage of the parent container size.

Then an appropriately written for_window rule would be able to apply
startup size hints on a per-application basis, e.g.:

    # Start pidgin windows at 20% of parent container width and full
    # screen height, if possible.
    for_window [class="Pidgin"] width='0.20' screen_height="100%"

Implementing the full blown layout specifications possible with CSS
would probably be overkill, but I am guessing we can go a very long way
without _all_ the bells and whistles.  Even with just _one_ size-related
hint, that depends on parent container size, we can do a lot of stuff.
