Quoting James Robertson (2012-04-21 01:41:14)
> I also noticed a similar issue in Gimp.  I am not sure if this is the
> same thing.
It is. However, in the case of Gimp, the problem is slightly different since
you are dealing with a floating window. In the future, i3 should actually pay
attention to the hints which are set on the window and resize it only in
correct multiples so that you don’t see the problem. In tiling mode the problem
remains, of course.

> Is there any way to resolve this?
Setting the client.background color to some color which comes as close to your
background as possible is the best thing you can do. See also this ticket,
which deals with this problem but in a slightly special context (borderless
windows):
http://bugs.i3wm.org/query/ticket/540
