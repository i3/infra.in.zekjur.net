Thanks, that’s good to hear.

> - Could we place all settings and (brief documentation as comments) inside the
> configuration template, like other big projects similarly do? (Apache, PHP,
> MySQL, etc). The fact it is all there, with default values, showing how it is
> behaving as of that moment kind of explains it better than an extensive
> document online.
No. I know a few other projects which do it like that, most notably
dovecot. I think that a huge file like that places a big burden on
people who look at the configuration file for the first time due to
information overload, therefore discouraging people to get into the
configuration.

Additionally, we have everything in the userguide because, being an HTML
document, it looks a lot better. You have nice formatting, graphics,
sections, different font sizes, etc. All of that is not achievable with
ASCII (at least not as good). I want to make the documentation as
appealing as possible.
