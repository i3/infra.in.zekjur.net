Hey guys,
I'd like to provide a patch for this ticket (
http://bugs.i3wm.org/report/ticket/807 ) but before I get started I'd like
to ask a couple of questions to start in the right direction.
I'm thinking about doing something like a scratchpad toggle which will
show/hide all scratchpad windows at once. This won't work on "all floating
windows" by default but on "all scratchpad windows". A window set to the
scratchpad is centered and resized the first time but not on future
"scratchpad show"s. So it's pretty easy to get the scratchpad windows
showing like you want and then toggle their visibility.
I think this is a good approach but I'm not sure. If this should work on
all the workspace floating windows, how can we distinguish the floating
windows being send to the scratchpad from windows already in the scratchpad?

So my questions are:
1) Is this the right approach?
2) If so, should the new command handle matching windows? (ie: bindsym
Mod1+Control+- [class="URxvt"] scratchpad toggle )


Rgds,
Marcos
