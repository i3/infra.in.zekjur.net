Hi Stefan,

Stefan Breunig <breunig at uni-hd.de> writes:
> – why do the input fields become red when focused? (I believe it’s  to 
> indicate that the input is wrong, but it still feels strange to be told 
> “you are wrong” right away)
Yes, it’s to indicate that they are required and you haven’t filled in
anything yet. That’s the default behavior of bootstrap, so if you care,
please send a patch against bootstrap’s css.

> – The logfile page could simply mention “i3-dump-log | gzip -9c > 
> /tmp/i3.log.gz” and link to the debugging page under "More info" or so, 
> since that’s the one command needed. Downside is that people might not 
> learn about how to obtain backtraces by chance.
Unfortunately, we cannot do that. i3-dump-log only works in debug
versions of i3, not in the release version (intentionally). A proper
error message which explains this in clear terms has been added in i3
v4.3, but that version is too new to assume that everybody has it
(granted, we do only support 4.3, but people might lie about the version
because they’re annoyed by the form, so let’s give it some time).

> – I always write the bug reports first, and then the title because I 
> made up my mind/organized my thoughts. For me, this usually results in 
> a more concise title. You could try to move the title down as well, but 
> call it “Summary” so it’s not so awkward.
Interesting idea. Are there any other opinions on that? I’m inclined to
call the field "one-line summary" and move it to the bottom, unless
someone comes up with a good reason against that.
