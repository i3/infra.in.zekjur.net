Things I noticed:

– why do the input fields become red when focused? (I believe it’s  to 
indicate that the input is wrong, but it still feels strange to be told 
“you are wrong” right away)
– The logfile page could simply mention “i3-dump-log | gzip -9c > 
/tmp/i3.log.gz” and link to the debugging page under "More info" or so, 
since that’s the one command needed. Downside is that people might not 
learn about how to obtain backtraces by chance.
– I always write the bug reports first, and then the title because I 
made up my mind/organized my thoughts. For me, this usually results in 
a more concise title. You could try to move the title down as well, but 
call it “Summary” so it’s not so awkward.
