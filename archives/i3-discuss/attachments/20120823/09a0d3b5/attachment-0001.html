The 'scratchpad show' command will only look at the scratchpad<br>
pseudo-workspace for windows to show. This explains what you are seeing.<br>
I don’t think it’s a bug, but the functionality could be enhanced to<br>
cover your use case. In case you’re interested, please file a feature<br>
request at <a href="http://bugs.i3wm.org/" target="_blank">http://bugs.i3wm.org/</a><br>
<br>
If you want to avoid this effect, you can use the scratchpad show<br>
command with criteria, like this:<br>
<br>
    bindsym Mod4+c [instance="cmus"] scratchpad show<br>
<br>
This will work on any workspace, because the window is found by checking<br>
its instance, *then* calling scratchpad show on it.<br>
