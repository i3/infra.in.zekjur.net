Hey Bruno -- I'm not a guru, but I'll share my story since I'm just a<br>
little ways ahead of you. And I've been wanting to pipe up and shower<br>
Michael et al with gratitude and praise for what so far looks like the<br>
biggest workflow improvement I've found in years. THANK YOU!!<br>
<br>
My suggestions:<br>
<br>
* Start by blowing away a toy machine and experiment there. Or I<br>
suppose you could be careful and just try to set up as a different<br>
user on your main machine. My i3/GNOME integration attempt didn't go<br>
well (but maybe it's commonly done). I'm enjoying having left GNOME<br>
behind now anyway. So i3/X is all that's needed!<br>
<br>
* Find someone's dotfiles online who's doing something similar. You<br>
probably just need .xinitrc and .i3/config, (and maybe .i3status) for<br>
starters. Mine have the bulk of the config (nm-applet, udiskie, font<br>
setting) in .i3/config<br>
(<a href="https://github.com/MicahElliott/dotfiles/blob/master/.i3/config" target="_blank">https://github.com/MicahElliott/dotfiles/blob/master/.i3/config</a>) as I<br>
found some things didn't take by just putting into .xinitrc.<br>
<br>
* You can bind keys easily enough with xev, and xmodmap. I used to use<br>
a crazy xkeycaps (IIRC) generated xmodmap to map keys all over the<br>
place, but all you need is probably a few lines like in your .xinitrc:<br>
<a href="https://github.com/MicahElliott/dotfiles/blob/master/.xinitrc" target="_blank">https://github.com/MicahElliott/dotfiles/blob/master/.xinitrc</a> . All<br>
you absolutely need in it is: exec ck-launch-session dbus-launch i3 .<br>
But I've also manually mapped volume controls and pm-suspend. I also<br>
call keychain just before firing i3.<br>
<br>
* Ease into the User's Guide <a href="http://i3wm.org/docs/userguide.html" target="_blank">http://i3wm.org/docs/userguide.html</a>, the<br>
one and only doc you really need. So nice. Work to memorize the<br>
bindings quickly.<br>
<br>
* Get familiarized with the non-GNOME tool set:<br>
<a href="https://wiki.archlinux.org/index.php/Common_Applications#Image_Viewers" target="_blank">https://wiki.archlinux.org/index.php/Common_Applications#Image_Viewers</a><br>
<br>
* Use lxappearance to change fonts (and theme if you like)<br>
<br>
* Rely on mod+r to quickly change things to your i3 config.<br>
<br>
* Decide a multi-monitor solution. i3 "just works" with zero config,<br>
but it's a little different. GNOME spreads a desktop across all<br>
monitors, but i3 treats each monitor as separate. I still haven't<br>
fully adjusted but have mapped even-numbered desktops to my top<br>
monitor and odds to laptop screen. This way I can hit ctrl-1-2 and<br>
later ctrl-3-4 to emulate the GNOME way. I'm mixing it up too, but<br>
still organize windows this way to large extent. I've bound mod+tab to<br>
quckly bounce from one desktop "back_and_forth" to the next. You can<br>
use arandr to config monitor settings and then export to a one-line<br>
xrandr script. I have 3 setups in ~/bin and can quickly change screens<br>
with ~/bin/xr1 (or xr2 or xr3).<br>
<br>
* Consider switching to Arch: rolling releases for bleeding edge of<br>
anything, easy to make your own packages, lots of light-weight package<br>
managers to choose from, have more control over what's on your<br>
machine, extensive and well curated wiki and forums, and seems to be a<br>
nice i3 cohort there. ArchBang was a decent install experience to get<br>
graphics, network, etc working OOTB (my head looked like yours after<br>
doing a vanilla Arch install :). And I stupidly let the installer<br>
choose my root partition at a small size and later tried to load big<br>
databases. So I'd recommend making / plenty big or just keeping /home<br>
on the same partition.<br>
<br>
* I couldn't give up my Alt key (not even close!), so mapped mod to<br>
Win key. (Right-Alt is my tilde/backtick, so esc is more reachable.)<br>
I'm mostly used to reaching for Win (Mod4) now, but I suppose<br>
conquering alt/win/ctrl/shift combos for everything is like mastering<br>
an instrument. I don't use arrow keys for anything (except stupid<br>
browsers and spreadsheets). And as a vim user I was COMPLETELY<br>
befuddled by the default jkl; mapping. i3 made it so easy to switch<br>
these things (how commonly do people change it to vim's hjkl?). And<br>
since I had to lose mod+h to the vim shift, I mapped mod+g to<br>
horizontal, which makes some sense being near the vertical v.<br>
<br>
* I use gimp often and i3's floating windows are working just fine for<br>
me. Get used to holding mod and using both mouse keys for quickly<br>
moving and resizing. Even non-floating windows benefit from<br>
mod+right-mouse.<br>
<br>
Superfluous background....<br>
<br>
I'd been using Ubuntu/GNOME for the last ~5 yrs, and using 10.04 LTS<br>
since its release. I'd done a lot of tweaking to the point that things<br>
were just getting too brittle (and stale). I figured I'd have to make<br>
a lot of changes to get going with Unity and now was as good a time as<br>
ever to simplify and reorganize. I decided to do the big-bang approach<br>
and change OS and WM all in one go. This has turned into a major time<br>
sink the last couple weeks but looking like it'll be worth it for<br>
another 5+ yrs of happiness (quite possibly on this same machine!).<br>
<br>
I started with xmonad and got it working pretty well on another Arch<br>
machine. Didn't even mind Haskell. Then put it on main system and hit<br>
a Haskell packaging problem, bloated my machine with ghc, got into a<br>
mess. And, was copying long recipes for xmonad/xmobar configs instead<br>
of doing simple things that i3 supports. Also had tried awesomewm and<br>
hit some bugs/surprises.<br>
<br>
Thanks to the simplicity and minimalism of i3 (and Arch) I can finally<br>
keep in my head where almost all my configs are and how to change<br>
things (well, my vim config is still a bit messy). I'm now on a small<br>
(and fast!) SSD and still have plenty of space. I can get new packages<br>
every day and all the tools I'm using are brand new. I love the tiny<br>
i3status and I've gone with the 1pixel window border that is working<br>
so nicely (mod+b is mapped to cycle *b*order modes; I like titles on<br>
some things like browsers). I've finally utilized every pixel, have<br>
custom tiny fonts on everything, and have reached WM nirvana. I could<br>
never get to this point of beautiful simplicity with GNOME.<br>
