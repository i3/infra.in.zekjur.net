</div></div>Hello.<br>
<br>
I guess your question is: „How to get this layout before launching VMD?”<br>
I think it's not possible (though I may be wrong,<br>
since I don't know if layout saving is implemented),<br>
but I can tell you how to get this layout after launching the program.<br>
<br>
1. With open terminal, switch to stacked mode („layout tabbed”)<br>
2. Launch VMD<br>
3. With one of the VMD's windows focussed, split horizontally („splith”)<br>
4. Focus the other VMD's window (probably „focus left”)<br>
5. Move the window into the split window (probably „move right”)<br>
<br>
Putting a for_window with splith command into your config<br>
could help to do this automatically for two windows,<br>
but that's a really ugly workaround.<br>
<br>
Before you start to fiddle with your configuration,<br>
can you tell why that program has to be launched from the terminal?<br>
If it doesn't require input and it doesn't output important data,<br>
I don't see the need for it to be launched from the terminal.<span class="HOEnZb"><font color="#888888"><br>
