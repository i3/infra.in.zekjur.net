      <p style="margin: 1.2em 0px ! important;">I use my laptop in three
        different environments with different randr outputs enabled.<br>
        I often switch between environments, and could not configure i3
        to place my workspaces on the desired outputs.<br>
        My randr configuration is updated automatically on udev drm
        change events with <a
          href="http://github.com/wertarbyte/autorandr">autorandr</a>.<br>
        Is there some way to specify dynamic output bindings ?</p>

        are pretty simple :</p>

            padding-left: 1em;">

              workspaces</li>
          </ul>
        </li>
        <li style="margin: 0.5em 0px;"><em>Docked</em>
          <ul style="margin: 1.2em 0px;padding-left: 2em;margin: 0px;

      <p style="margin: 1.2em 0px ! important;">This is already a hack
        because the configuration describes the <em>Docked</em>
        configuration.<br>
        It incidentally works when <em>Undocked</em> because there is
        only one output, and all workspaces are mapped to it.</p>
      <p style="margin: 1.2em 0px ! important;">It does not work at all
        for my presentation.</p>
      <p style="margin: 1.2em 0px ! important;">One solution that would
        suit me would be to extend the <code style="font-size: 0.85em;
          font-family: Consolas,Inconsolata,Courier,monospace;margin:
          0px 0.15em; padding: 0px 0.3em; white-space: pre-wrap; border:
          1px solid rgb(234, 234, 234); background-color: rgb(248, 248,
          248); border-radius: 3px; display: inline;">workspace
          <x> output</code> command to support multiple outputs in
        order of preference.<br>
        for example, <code style="font-size: 0.85em; font-family:
          Consolas,Inconsolata,Courier,monospace;margin: 0px 0.15em;
          padding: 0px 0.3em; white-space: pre-wrap; border: 1px solid
          rgb(234, 234, 234); background-color: rgb(248, 248, 248);
          border-radius: 3px; display: inline;">worspace 0 output(s)
          VGA1 eDP1</code> would place workspace 0 on the projector when
        available, and on eDP1 otherwise.</p>
      <p style="margin: 1.2em 0px ! important;">Do you find this useful
