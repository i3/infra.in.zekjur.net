PROCESS=${1}
STATE=$(cat <i class="moz-txt-slash"><span class="moz-txt-tag">/</span>proc<span class="moz-txt-tag">/</span></i>$(pgrep ${PROCESS})/stat | awk -F' ' '{print $3}')

[[ "${STATE}" = "T" ]] && pkill -SIGCONT ${PROCESS} || pkill -SIGSTOP
${PROCESS}

        <br>
        <br>
            bindsym $mod+p exec --no-startup-id /some/path/stopcont.sh
        <processname>
        <br>
        <br>
        Script:
        <br>
        ===========
        <br>
        #!/usr/bin/env bash
        <br>
        PROCESS=${1}
        <br>
        FILE="/tmp/${PROCESS}.signal"
        <br>
        <br>
        if [[ -f "${FILE}" ]]; then
        <br>
          rm ${FILE}
        <br>
          pkill -SIGCONT ${PROCESS}
        <br>
        else
        <br>
          touch ${FILE}
        <br>
          pkill -SIGSTOP ${PROCESS}
        <br>
        fi
        <br>
        ===========
        <br>
        <br>
        This will, of course, assume that the process isn't paused by
        anything
        <br>
        else as that would make it go out of sync. A better and more
        robust way
        <br>
        would be to write the script such that it instead uses the
        information
        <br>
        of /proc/<pid>/stat to check whether the process is
        currently paused. If
        <br>
        you use a higher-level language such as Python, you can do this
        nicely
        <br>
        instead of manually parsing the output. See [1] for some more
        information.
