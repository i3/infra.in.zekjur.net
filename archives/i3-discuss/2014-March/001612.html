btw, i3blocks does handle events.
If you set for instance signal=10 to a block, it will be updated when
i3blocks receives a SIGUSR1 (you can take a look at the default "volume"
block, and try "amixer -q set Master 2dB+ unmute && killall -USR1
i3blocks").
Also, if your script does not check the BLOCK_BUTTON env variable, clicking
on the block will result in re-updating it (it's particulary useful for the
network block, when connecting to a SSID or (un)plugging ethernet and want
to have the IP update right away).
