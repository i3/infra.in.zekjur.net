Excerpts from Alex V. Koval's message of 2012-02-03 06:29:42 +0000:
>   1) Still I think it would be very good for the Alt-Tab support.
I find the concept of alt-tab horrible. In its usual implementation (like it is
implemented in Windows, Mac OS X, GNOME, …), it either requires you to remember
where focus was before, or you end up in the wrong window. Now, most of the
mentioned implementations also display some kind of window list to make this
more deterministic, but that requires additional hand-eye-coordination, which
i3 attempts to get rid of (mostly).

Therefore, I don’t want to have an alt-tab-like feature. Most users get used to
using the direction keys just fine.

>   2) Smart container auto-split. That feature may be optional. Now,
> I think I will stay with the following: after new app open, I *always* should
> specify manually how the next application will split the current container.
> Lets see if I can get this habit. Because what happens now is that a lot
> of windows open (chat windows, etc) usaully in wrong direction, and I almost
> always have to relocate or re-split the container and relocate window
> back to correct location.
Hm, I usually set my workspaces to stacked layout (unless I’m using very big
monitors) and then break out of that (by using focus parent and opening a new
window) when necessary. For more advanced rules, like splitting based on the
amount of windows / space used, I would recommend writing external scripts, but
I’m not quite sure how exactly that could be done from the top of my head :).

If we forgot to reply to any of your points, please let us know. The list was
pretty long :).
