Hi Martin,

Quoting Martin Kraus (2012-08-21 14:29:11)
> I've noticed with the scratchpad window that if I open it on a workspace and
> then move to a different workspace I cannot bring it up again. I need to
> return the the original workspace and put it into background to open it on the
> new workspace again. This kind of negates the purpose of the scratchpad.
> Is this a feature, bug or just my missunderstanding?
The 'scratchpad show' command will only look at the scratchpad
pseudo-workspace for windows to show. This explains what you are seeing.
I don’t think it’s a bug, but the functionality could be enhanced to
cover your use case. In case you’re interested, please file a feature
request at http://bugs.i3wm.org/

If you want to avoid this effect, you can use the scratchpad show
command with criteria, like this:

    bindsym Mod4+c [instance="cmus"] scratchpad show

This will work on any workspace, because the window is found by checking
its instance, *then* calling scratchpad show on it.
