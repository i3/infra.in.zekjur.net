Hi Jean,

Quoting Jean Dupont (2012-08-24 14:38:51)
> I discovered is that you can not specify on which side you want to split
> the window. It is always the right (or lower) side.
> 
> My idea:
> Leave it that way as default, but if the user also presses another key
> (down, up, k, l, ...) that indicates a direction i3 use this side.
> 
> Keys pressed = result:
> $mod+v = new Window under the existing one
> $mod+v+down = the same result
> $mod+v+up = now the window appears above the old one
> Something equal for splitting horizontally.
This introduces a lot of new keybindings, configuration changes,
documentation changes, friction on upgrade for users and I don’t see a
clear benefit.

The splitting works by creating a new split container and placing the
current window in it. We always append new windows to the end of the
node list. This is the cause for your observation that new windows
appear "below" existing windows. Since from left to right and top to
bottom is very natural, I don’t see an advantage in introducing an
additional feature which puts new windows at the top of the node list.

Best regards,
Michael
